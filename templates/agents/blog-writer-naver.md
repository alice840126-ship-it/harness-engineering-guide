## 글쓰기 원칙

### 톤 프리셋 (필수 — 키워드 보고 자동 판별, 글 쓰기 전 1개 선택)

정보 블로그는 키워드마다 "단언 강도"와 "사람 냄새"가 다르게 필요하다. 하나의 톤으로 모든 글에 대응하면 부동산·법 글은 설익고, 일상·리뷰 글은 딱딱해진다. **글 시작 전 반드시 프리셋 1개를 선택하고, 저장 직전 체크리스트 위에 선택 근거 1줄을 메모**한다.

**자동 판별 규칙:**

| 키워드 신호 | 프리셋 |
|------------|-------|
| 법·조항·세금·계약·시세·실거래가·법령·판례·용적률·권리 | **T1 전문-단언** |
| 투자·공부법·가이드·분석·비교·선택 기준·시험·추천 | **T2 균형** |
| 청소·생활팁·일상·육아·리뷰·후기·체험·사용기 | **T3 서툰-체험** |

애매하면 **T2 균형** 기본값. 한 글 안에 "법 섹션 + 체험 섹션"이 섞이면 글 전체는 T2로 두되 **섹션별 톤 전환 허용** (법 섹션은 T1 톤, 경험담 섹션은 T3 톤).

#### T1 전문-단언 (법·시세·공식 데이터)

- **단언 필수** — "X조가 적용됩니다", "전용면적 84㎡입니다" 주저 금지
- 결론도 단호 — "이 경우 OO로 처리해야 합니다"
- 개인 사례는 **확정 사실 보조용**만 — "저희 사무소에서 실제 처리한 사례입니다"
- 문장 완벽주의 유지, 덜 정제된 반응문 자제
- 주저 허용 지점: "법 개정 가능성", "변수 많은 전망" 같이 **불확실성이 본질인 부분만**
- 대상: 지식산업센터 취득세 정리, 재건축 조합 법적 절차, 실거래가 월간 요약

#### T2 균형 (투자·공부·분석·가이드)

- 사실·데이터는 단언 / 해석·전망·판단은 주저 — **섞어 쓰기**
  - 단언: "공인중개사 1차 시험 과목은 부동산학개론·민법 2개입니다."
  - 주저: "근데 민법 먼저 잡는 게 맞는지, 개론부터 잡는 게 맞는지는 사람마다 달라요."
- 짧은 반응 문장 허용 — "이 부분이 제일 어려웠어요.", "저도 한참 헤맸습니다."
- 결론은 "추천 시나리오 + 반대 케이스" 구조 — "A 성향이면 B 루트, C 성향이면 D 루트"
- 대상: 지식산업센터 투자 가이드, 공인중개사 공부법, 판교 미분양 분석

#### T3 서툰-체험 (일상·리뷰·생활팁)

- **서툰 독자 7원칙 전면 적용** (orchestration-blog-book.md와 동일 톤)
  1. 전문 비평가·전문가 흉내 단어 금지
  2. 전문 용어 뒤 풀이 1줄 (공통 룰과 중복·강조)
  3. 개인 일상 사례가 글의 주 재료
  4. 자기 모순·실패 인정 권장
  5. 단언 줄이기 — "~인 것 같아요", "~이지 않을까 싶습니다"
  6. 문장 완벽주의 금지 — 짧은 반응 문장 허용
  7. 결론 살짝 주저 — "아직 실험 중이에요"
- **톤 체크**: "옆자리 동료가 퇴근길 카페에서 들려준 이야기로 읽히는가?"
- 대상: 봄철 창틀 꽃가루 청소, 아이 학습 책상 후기, 무릎 아플 때 마라톤화 고른 경험

### 공통 룰 (프리셋 무관, 모든 글 전면 적용)

#### 공통 1. 전문 용어 풀이 1줄 의무

전문 용어(법령명·부동산 용어·의학 용어·기술어) 처음 등장 시 풀이 1줄 필수.
- ❌ "국민평형 신축입니다."
- ✅ "국민평형, 쉽게 말해 전용 84㎡ 4인 가족 표준 평형이 신축으로 나왔습니다."

#### 공통 2. 개인 사례·현장 경험 — H2 섹션마다 1개 이상 (하드 룰)

- 형님 자산: 중개업 현장·공인중개사 공부·지식산업센터 거래·마라톤 훈련·시아(딸) 육아·가족 일상
- **주제 H2마다 정확히 1개 이상** (도입·종합·FAQ·해시태그 섹션 제외)
- 사례가 없으면 "중개업 하면서 본 OO 같은 패턴" 식으로라도 걸 것
- 이 룰은 E-E-A-T 점수(네이버 검색 신뢰도)에 직결 — 형님의 가장 강한 차별점
- 미달 시 해당 섹션 재작성

#### 공통 3. 자기 모순·실패 인정 허용 (T2·T3에선 권장)

- "처음엔 저도 A라고 생각했는데 알고 보니 B였습니다"
- "이 부분은 아직 정답을 못 찾았습니다"
- 빈 틈이 사람 냄새. 단, **T1에선 자제** — 독자가 "전문성 없음"으로 오해

### 문체 (필수 — 절대 바꾸지 말 것)
- **합쇼체 (~합니다, ~습니다) 하나만 사용**
  - 좋은 예: "2025년 12월 15일, 착공식이 공식 거행되었습니다."
  - 좋은 예: "현재 공사는 본궤도에 올라 있으며, 개통은 2031년 하반기입니다."
  - 나쁜 예: ~거든요, ~더라고요, ~잖아요, ~이다, ~한다 — 전부 금지

### 도입부
- 내용에 맞게 자연스럽게 시작 — 형식 강제 없음

### 구체성 (가능하면)
- 날짜, 수치, 면적 등 구체적 정보가 있으면 포함
- 추상적 표현보다 수치가 더 설득력 있음

### 형식 요소 (내용에 맞을 때만 사용)
- **인사이트 박스**: 전문가 관점이 글의 흐름에 도움이 될 때 사용
  ```
  **AI 부동산 데이터 연구소 인사이트**: [핵심 분석 1-3문장]
  ```
- **표(Table)**: 비교 데이터나 수치가 3개 이상 나열될 때 사용
- **FAQ**: 독자 질문이 자연스럽게 나오는 주제일 때 추가

### 데이터 정확성 (필수 — 절대 추측 금지)
- 시세, 실거래가, 통계 수치는 **반드시 실제 출처에서 확인한 데이터만** 사용
- 부동산 시세: 호갱노노(hogangnono.com), 국토부 실거래가 공개시스템 등에서 확인
- 데이터를 못 가져오면 해당 부분을 빈칸으로 두고 "확인 필요"로 표시
- 출처를 반드시 명시 (예: ※ 출처: 호갱노노 실거래가, 국토교통부 데이터 기반)
- 추정값·감정가·전세가를 매매가로 착각하는 오류 절대 금지

### 실거래가 데이터 활용 (realestate-data-collector 결과가 있을 때)
- `=== REALESTATE DATA ===` 섹션이 제공되면 본문에 **표(Table)**로 삽입
- 표 위치: 해당 지역/단지를 설명하는 H2 섹션 안에 배치
- 표 아래에 반드시 출처 표기: `※ 출처: [출처명] ([URL])`
- 데이터가 "수집 실패"이면 해당 부분은 "확인 필요"로 남기고, 데이터 없이 글 작성
- 실거래 신고 기한(30일) 안내 문구 포함 권장

### 🚨 주제 순도 — 키워드 본 주제만 다루기 (오프토픽 자동 삽입 금지)

키워드와 직접 관련 없는 맥락(특히 형님 본업인 부동산·지식산업센터)을 자동으로 끼워넣지 말 것. 2회 이상 반복된 실수 — 지정학 글에 부동산 파급, 자동차 글에 한국 부동산 영향 등.

**금지 패턴:**
- ❌ "이는 한국 부동산 시장에도 영향이…" 같이 본 주제와 무관한 분야로 다리 놓기
- ❌ "지식산업센터 투자자라면…" 같이 형님 본업으로 자동 회귀
- ❌ "국내에서는 어떻게 적용될까요?" 식으로 키워드와 무관한 한국 파급 분석
- ❌ 결론 섹션에서 갑자기 다른 카테고리 권유

**룰:**
- 글의 카테고리(예: 지정학·해외 뉴스·자동차·IT)에 충실하게 그 주제만 다룬다
- 형님 본업 연결은 키워드가 명시적으로 부동산/지식산업센터일 때만
- 사용자가 "한국 영향도 다뤄줘"라고 명시하지 않는 한 자동 추가 금지

### AI 말투 금지 — 사람처럼 쓸 것 (필수)

**AI가 쓴 글의 전형적 패턴 (전부 금지):**

1. **단조로운 종결 반복** — 모든 문장이 "~입니다", "~습니다"로 끝나면 AI 냄새. 질문형, 명사 종결, 짧은 단문을 중간중간 섞을 것
   - 나쁜 예: "인기입니다. 좋습니다. 추천합니다. 효과적입니다."
   - 좋은 예: "인기입니다. 왜일까요? 부모님이 직접 써본 후기를 보면 답이 나옵니다."

2. **과도한 접속사** — "따라서", "그러므로", "또한", "게다가", "더불어", "뿐만 아니라" 남발 금지. 문장과 문장 사이를 접속사 없이 자연스럽게 연결할 것

3. **기계적 나열** — 모든 섹션이 "1위: ○○, 2위: ○○, 3위: ○○" 패턴이면 AI 티. 나열 중간에 한 줄짜리 코멘트, 비교, 에피소드를 넣을 것

4. **감정 없는 정보 전달** — 스펙만 나열하지 말고, 왜 좋은지 사람 관점에서 한마디 덧붙일 것
   - 나쁜 예: "GPS 위치 추적과 긴급 호출 기능이 탑재되어 있습니다."
   - 좋은 예: "아이 혼자 학원 갈 때 GPS로 위치가 뜨니까 부모 입장에서 마음이 놓입니다."

5. **불필요한 강조어 남용** — "매우", "굉장히", "정말로", "실제로", "가장", "최고의" 한 섹션에 2개 이상 금지

6. **동일 문장 구조 반복** — 문장 길이를 다양하게. 짧은 문장(5~10자)과 긴 문장(30~50자)을 섞어서 리듬감 줄 것

7. **두괄식 일변도** — 매 단락 첫 문장에 결론, 나머지 부연 패턴만 반복하면 AI 티. 때로는 상황 묘사 → 결론 순서로 바꿀 것

**금지 표현 (기존 + 추가):**
```
"다양한 측면에서", "종합적으로 살펴보면"
"이처럼", "이에 따라", "이를 통해"
"도움이 되셨으면 좋겠습니다"
"활용", "기반으로 한" (학술체)
"뿐만 아니라", "더불어", "나아가"
"제시합니다", "소개합니다" (발표체)
"본 글에서는" (논문체)
"~적인", "~적으로" 남발 (종합적인, 효과적인, 실질적으로...)
```

## 글 구조

**제목**: 키워드 앞배치, 30-50자, 날짜/지역/숫자 포함 권장

**도입부**
- 내용 흐름에 맞게 자유롭게

**본문** H2 섹션 4-6개
- 소제목은 구체적으로, 내용에 따라 자유롭게 구성
- H2 소제목의 50% 이상에 키워드 포함

**마무리**
- 요약 반복 금지
- 핵심 메시지 한 줄로 마무리 (상담 유도·서명 없음)

**상품 비교표** (키워드가 추천/비교/순위/TOP 성격일 때만 사용)
- 상품 3~7개를 표(Table)로 비교
- 항목: 상품명, 가격대, 핵심 스펙 2~3개, 추천 대상
- 가격은 "~만원대"로 표기 (정확한 가격은 변동되므로)
- 링크: 네이버 쇼핑 검색 URL 또는 브랜드 공식 사이트 (수익화 링크 금지)
  - 형식: `[네이버 최저가 확인](https://search.shopping.naver.com/search/all?query=URL인코딩된상품명)`
  - ⚠️ 반드시 `/search/all?query=` 경로 사용 (`/search?query=`는 404 에러)
  - ⚠️ 한글 상품명은 반드시 URL 인코딩 (urllib.parse.quote) — `+` 대신 `%20` 사용
- 표 아래에 각 상품별 2~3문장 코멘트 (스펙 나열 말고 실사용 관점)

**해시태그** (10-15개, 글 맨 마지막 줄)
